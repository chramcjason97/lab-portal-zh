# lab-portal-zh

本仓库用于归档和发布多个独立的 HTML 页面。  
这些页面各自实现特定的功能或展示，彼此之间无域名或网站关联。  
仓库内容以静态页面为主，适合直接通过 GitHub Pages 或其他静态托管服务访问。

## 项目简介

`lab-portal-zh` 是一个轻量级的页面集合仓库。  
每个 HTML 文件代表一个独立的“实验页面”或“工具页面”，不依赖外部框架或后端服务。  
仓库主要目的为：

- 归档已完成或实验中的 HTML 页面
- 提供可直接运行的静态页面副本
- 方便版本管理与历史回溯

## 目录说明

```
.
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，命名应尽量反映其功能或编号。
- `assets/`：存放所有页面可能共享的静态资源。建议每个页面尽量自包含，减少跨文件依赖。

## 页面归档说明

- 每个页面均为独立、完整的 HTML 文件（可包含内联样式与脚本）。
- 页面之间无跨域或跨页面通信需求。
- 归档时请确保页面在本地或静态服务器上可正常打开。
- 若页面依赖外部资源（如 CDN 库），请在文档中注明。

## 维护说明

- 新增页面：在 `pages/` 目录下添加新 HTML 文件，并更新入口页面（如有）。
- 修改页面：直接编辑对应 HTML 文件，建议附带简要提交说明。
- 删除页面：移除对应文件，并清理入口页面中的链接。
- 不推荐在仓库中存放动态内容或需后端支持的页面。

## 使用方式

1. 克隆本仓库到本地。
2. 直接使用浏览器打开 `pages/` 下的 HTML 文件查看效果。
3. 或启用 GitHub Pages（Settings → Pages → Source），选择主分支根目录即可在线访问。
